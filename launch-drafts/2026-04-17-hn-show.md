## Body (post as first comment immediately after submitting)

WikiMem is a CLI and web IDE that compiles your sources — PDFs, audio, video, spreadsheets, URLs, and 38 OAuth/API connectors — into a structured wiki of entity, concept, and source pages with [[wikilinks]] between them. The output is plain markdown, Obsidian-compatible, and git-checkpointed.

I built it because RAG kept being the wrong shape for personal knowledge. Retrieval gives you paragraphs at query time. A wiki gives you structure you can actually read, cite, and hand an LLM. Karpathy posted the pattern on April 7. I started that night and shipped the 1.0 ten days later.

Three automations run the vault: an ingest pipeline (file watcher → processor → LLM compile → git commit), a scraper (RSS / GitHub queries / URLs on cron), and an Observer that scores every page on 24 points across five dimensions (coverage, consistency, cross-linking, freshness, organization), then fixes the weakest ones. The Observer is budget-capped — max 3 page rewrites and 8 contradiction-pair comparisons per run, roughly 11 LLM calls and a $2 ceiling — so a vault with 10,000 pages does not produce a 10,000-dollar bill.

The stack is boring on purpose: TypeScript, Express, D3, simple-git, node-cron, chokidar, pdf-parse, mammoth, ffmpeg, Whisper, the Anthropic / OpenAI / Ollama SDKs. 23K lines, 87 tests. MIT.

The part I care about most is the MCP server. 19 tools (`wikimem_search`, `wikimem_read`, `wikimem_ingest`, `wikimem_ask`, `wikimem_run_observer`, `wikimem_sync`, etc.). Add five lines to `.mcp.json` and every Claude Code conversation reads your compiled knowledge directly — no copy-paste, no context-stuffing.

Runs fully local with Ollama (no API keys, no network). `raw/` is gitignored by default; `config.yaml` is gitignored so keys never commit.

Try it:

    npx wikimem init my-wiki
    cd my-wiki
    npx wikimem serve

Open http://localhost:3141 — file tree, WYSIWYG editor, D3 knowledge graph, time-lapse scrubber (watch the wiki grow commit by commit), and pipeline view.

Happy to discuss: wiki vs RAG for personal knowledge; the Observer's budget model; the 3-layer architecture (raw/wiki/AGENTS.md); why the connector layer is MCP-native rather than a proprietary integration protocol; and how much of this was written by 22 Claude Code agents running in parallel (most of it).
